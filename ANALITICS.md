# Итоговый проект: Анализ данных GH репозиториев по сохранённым фильтрам

## Идея сервиса и источники данных
Я хочу написать сервис для анализа данных из различных источников по технологиям. Например, это может быть анализ данных из GitHub-репозиториев, Reddit, блогов с RSS-лентами и так далее. Начнем с GitHub-репозиториев.

## Эндпоинт 1: Получение данных
Изначально сервис должен иметь несколько эндпоинтов. Первый — это получение данных по выбранным технологиям и фильтрам. Фильтры можно задать либо непосредственно в запросе, либо использовать сохраненные в хранилище приложения.

Этот эндпоинт возвращает данные в формате JSON по заданным критериям. Например, если заданы фильтры: Python-репозитории с оценкой больше 150 звезд, временем жизни с марта 2023 года по настоящее время и >100 подписчиков, — мы должны найти в локальном хранилище или на GitHub все подходящие репозитории. Пользователю отдается список с именем, количеством звезд, временем жизни и количеством контрибьюторов.

    Задача со звездочкой: добавить фильтры, которые не поддерживает сам GH

## Эндпоинт 2: Управление фильтрами
Второй эндпоинт предназначен для задания пользовательских фильтров. Пользователь может указать, что ему нужны Python-репозитории с определенным количеством звезд, временем жизни, числом контрибьюторов и так далее. Набор этих фильтров четко формализован и проверяется при вводе. Также пользователь имеет возможность сбросить все свои настройки или удалить конкретный фильтр по имени.

## Эндпоинт 3: Отчеты и аналитика
Пользователь может сгенерировать отчет по своим фильтрам и метрикам. Например, рассчитать среднее количество звезд по конкретному источнику за несколько месяцев или сравнить репозитории по какому-то параметру (количеству звезд или контрибьюторов). В результате он получает ссылку на график, сохраненный в S3, SFTP или локальном хранилище

## Эндпоинт 4: Добавление новых источников
Пользователь может предложить новый источник данных и отправить его на модерацию. Например, он может запросить получение данных из определенного блога. Запрос отправляется администраторам, которые рассматривают возможность добавления. Если источник одобряют и добавляют в общий доступ, пользователю начинают приходить данные по этому ресурсу.

## Фоновые задачи (Cron Jobs) (задача со звездочкой)
В системе предусмотрена Cron-задача, которая, например, раз в день обращается к источникам и собирает информацию по пользовательским фильтрам. Система проверяет все активные фильтры (Python, звезды, форки, GitHub, Reddit), опрашивает соответствующие источники, получает данные и сохраняет их в локальное хранилище.

## Получение данных из новых источников (задача со звездочкой)
Реализовать получение данных из источников, которые предлагают пользователи.
Например, кроме интеграции с GH, можно сделать интеграцию с reddit api. Тогда пользовательские фильтры нужно разбивать ещё и по источникам
